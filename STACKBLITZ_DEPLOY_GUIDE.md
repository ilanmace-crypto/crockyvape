# Деплой на StackBlitz - Пошаговая инструкция

## Шаг 1: Создание проекта на StackBlitz
1. Зайди на https://stackblitz.com
2. Нажми "Start from GitHub" (или "Import from GitHub")
3. Введи URL репозитория: `https://github.com/ilanmace-crypto/paradise-shop.git`
4. Выбери "Node.js" как тип проекта
5. Нажми "Import"

## Шаг 2: Настройка проекта
1. В файле `package.json` измени основной скрипт:
   ```json
   "scripts": {
     "start": "node server-replit.js"
   }
   ```

2. Установи зависимости:
   - В терминале (внизу) введи: `npm install sqlite3 express cors body-parser dotenv`

## Шаг 3: Сборка React приложения
1. В терминале введи:
   ```bash
   npm run build
   ```

## Шаг 4: Запуск проекта
1. В терминале введи:
   ```bash
   npm start
   ```

2. Сайт автоматически откроется в preview
3. Адрес проекта будет вверху: `https://paradise-shop.stackblitz.io`

## Шаг 5: Публикация проекта
1. Нажми "Share" (вверху справа)
2. Скопируй ссылку проекта
3. Любой может открыть сайт по этой ссылке

## Шаг 6: Проверка работы
1. API: `https://paradise-shop.stackblitz.io/api/health`
2. Товары: `https://paradise-shop.stackblitz.io/api/products`
3. Фронтенд: `https://paradise-shop.stackblitz.io`

## Преимущества StackBlitz:
- **Никогда не спит** - работает 24/7
- **Постоянный домен** - не меняется
- **Мгновенные обновления** - без перезагрузки
- **VS Code IDE** в браузере
- **SQLite работает** из коробки
- **Полностью бесплатно**

## Особенности работы:
- **WebContainers** - эмуляция в браузере
- **Автосохранение** всех изменений
- **Live reload** - сайт обновляется в реальном времени
- **GitHub синхронизация** - изменения сохраняются

## Возможные проблемы и решения:

**1. SQLite не работает:**
- Перезапусти терминал
- Проверь что файл `paradise-shop.db` создан

**2. Порт занят:**
- Измени порт в `server-replit.js`
- Используй порт 3000 или другой свободный

**3. Фронтенд не загружается:**
- Проверь что `npm run build` выполнен
- Проверь папку `build`

**4. API не отвечает:**
- Проверь консоль в терминале
- Убедись что сервер запущен

## Ограничения:
- **WebContainers** - не настоящий сервер
- **Ограниченная сеть** - некоторые API могут не работать
- **Память ограничена** - для больших проектов
